# 오답노트

신경써야할 부분은 

n, m 의 len 길이는 일정하게 n < m 이 아니여서 
유동적으로 변화되는 값이 되여야한다.

그래서 조건을

if n > m :  이라는 조건으로 걸고
n, m = m, n 
n_list, m_list = m_list, n_list 
이렇게 바꿔주면 로직을 굳이 바꾸지않아도 항상 긴값에서 작은값의 계산이 이루어짐

로직에서는 

내가 놓친 부분은 딱 처음 for문의 범위이다.
for 문 범위를 지정할때 단순히 range(n) or range(m) 을 하게되면
고정이 되버리기 때문에 둘의 값이 바꼈을때는 범위를 벗어나게 된다.
그래서 변동되도 변동되지않는 수가 필요하다
긴거에서 짧은거를 뺀값이 일정하다는 부분을 이용하는 거다

이문제에서는 

for i in range(m - n + 1)  
만약 m = 5 n = 3 이라면 
5 - 3 + 1 = 3 이다 
여기서는 조건이 항상 n 은 작은값 m은 큰값으로 유지되기때문에
m - n + 1 로 기존 n 이라는 수를 유지시킬 수 있다.


이렇게만 정해주면 

for i in range(m - n + 1):

    for j in range(i, i + n):

    이렇게만 범위를 지정해줘도 일정하게 한칸씩 증가된 상태로 순회시킬수 있다.
